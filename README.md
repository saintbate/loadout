# Loadout

A web app that helps builders pick the right combination of AI tools (MCP servers, CLIs, APIs, libraries) for what they want to build, and produces a detailed plan-view recipe with reasoning per tool choice.

## Stack

- Next.js 15 (App Router) + TypeScript
- Neon Postgres + Drizzle ORM
- Clerk auth
- Anthropic API (`claude-sonnet-4-6`) — structured tool-use planner
- Tailwind + shadcn/ui primitives, Shiki for code highlighting
- Vercel (deploy target)

## Setup

```bash
npm install
cp .env.local.example .env.local
# fill in DATABASE_URL, Clerk keys, ANTHROPIC_API_KEY, ADMIN_CLERK_USER_ID
```

### Environment variables

| Var | Where to get it |
| --- | --- |
| `DATABASE_URL` | Neon dashboard → connection string (pooled, `sslmode=require`) |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk dashboard → API Keys |
| `CLERK_SECRET_KEY` | Clerk dashboard → API Keys |
| `ANTHROPIC_API_KEY` | console.anthropic.com → API Keys |
| `ADMIN_CLERK_USER_ID` | Your Clerk `user_id` — gates `/admin` |

### Database

```bash
npm run db:push       # push the Drizzle schema to Neon
npm run db:generate   # (optional) generate SQL migrations under ./drizzle
npm run db:studio     # browse the DB
npm run db:seed       # upsert ~37 tools and ~6 starter recipes
```

### Dev

```bash
npm run dev           # http://localhost:3000
curl localhost:3000/api/health   # confirms DB connectivity
```

## Routes

| Route | What it does |
| --- | --- |
| `/` | Goal input → plans a recipe via Anthropic tool-use, redirects to its slug |
| `/recipe/[slug]` | Plan-view: steps, tool roles, rationale, alternatives, code, trust badges |
| `/browse` | All non-deprecated recipes — filter by tag, search by goal, sort popular/recent |
| `/tool/[slug]` | Tool detail — capabilities, compatibility (works_with/replaces/…), recipes using it |
| `/submit` | Auth-gated. Submit a recipe via goal (runs planner) or pasted JSON. Lands as `community` |
| `/admin` | Gated by `ADMIN_CLERK_USER_ID`. Promote `community` → `verified`, demote, deprecate |
| `/api/plan` | POST {goal} → plan JSON (used internally by homepage and `/submit`) |
| `/api/health` | DB connectivity check |

## Trust signals

Each step in a plan carries a `trust_signal`:

- **`verified`** — every tool is in the directory and `status="active"`
- **`untested`** — every tool is in the directory but at least one is `unstable`/`deprecated`
- **`proposed`** — at least one tool isn't in the directory yet (`proposed_tool: true`)

Recipes themselves have a separate `status`:

- **`draft`** — generated by the homepage planner, not yet submitted
- **`community`** — submitted via `/submit`, awaiting review
- **`verified`** — promoted by an admin who confirmed it works
- **`deprecated`** — hidden from `/browse`
